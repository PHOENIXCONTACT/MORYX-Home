# MORYX-Home

This repository is the central information hub for MORYX developers

## Guidelines

This repository serves as the central information hub for MORYX developers. Here you will find guidelines, tutorials, architecture and component design principles, as well as process documentation covering release and ticket creation workflows.

## Contents

- **Component Design Guidelines**  
  Learn how to structure and define components within MORYX, including namespace conventions and recommended patterns.  
  *See*: [`development/architecture/components.md`](development/architecture/components.md)

- **Release Process**  
  Step-by-step instructions on how to create releases and manage versioning for both Github and Gitlab repositories.  
  *See*: [`processes/release_process.md`](processes/release_process.md)

- **Ticket Creation Guidelines**  
  Checklist to help you document tickets clearly, define acceptance criteria, and organize stories, epics, and accounts.  
  *See*: [`processes/ticket_creation.md`](processes/ticket_creation.md)

- **Tutorials**
  - How to create a MORYX Web UI (Razor Class Library setup and integration)  
      *See*: [`tutorials/01_HowToCreateYourUI.md`](tutorials/01_HowToCreateYourUI.md)
  - Add language support to your UI using translation JSON files and the required services  
      *See*: [`tutorials/addLanguageSupport.md`](tutorials/addLanguageSupport.md)

## Getting Started

This repository is for documentation only. Browse the Markdown files in this repository to find information relevant to developing with MORYX, setting up your development environment, and following project processes.
